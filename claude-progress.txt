#### Feature #204: UI bug — sidebar skeleton reload on article click ✅
- **Root cause**: TocSidebar was rendered inside `page.tsx` (Server Component).
  Every article navigation re-rendered the page, which triggered `loading.tsx`
  (Suspense fallback) showing the full skeleton including the sidebar.

  instead of receiving it as a prop from the page. Uses `useMemo` to parse

  it internally via URL).

  `loading.tsx` — they now live in the persistent layout.

  skeleton when year or locale changes (different content).
- Biome lint: 0 new errors, TypeScript tsc: 0 errors
- All pages return HTTP 200

#### Feature #198: Compare page scroll — sticky controls ✅
- **Root cause**: Classic CSS flexbox gotcha. The scrollable content area had
  `flex-1 overflow-y-auto` but flex items have an implicit `min-height: auto`
  which prevents them from shrinking below content height. The content expanded
  beyond the container, causing page-level scroll instead of container scroll.
- **Fix**: Added `min-h-0` to the scrollable content div to override the
  default `min-height: auto`. This allows `overflow-y-auto` to activate
  within the flex container, keeping the controls panel fixed.
- Controls (version selectors, summary badges, filters, navigation) stay
  visible while diff content scrolls independently.
- Biome lint: 0 new errors, TypeScript tsc: 0 errors
- Compare page returns HTTP 200
